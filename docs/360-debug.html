360M Debug

Phone-safe diagnostic page. It does not replace the main tester. It checks whether SmolLM2 360M fails because of backend, dtype, prompt format, or output extraction.

Status: Idle

Setup

Diagnostic controls

Phone-safe default: 360M / WebGPU / q4 / 16 tokens. Do not use omitted dtype, fp16, or fp32 on phone unless you expect a possible browser crash.

Why this exists

What we are isolating

The main tester worked with 135M. The 360M path produced blank output in patched runners and gibberish in the restored runner. This page runs one small generation at a time and prints the raw returned object so we can see whether the issue is dtype, backend, chat formatting, or output extraction.
