# Building

## Automatic GitHub build

Push to the `local-ai-workbench` branch or run **Build Android APK** manually.

The workflow:

1. Installs Java, Android SDK, NDK, CMake, and Gradle.
2. Builds the Android plugin and its ARM64 native library.
3. Fetches a pinned llama.cpp commit through CMake.
4. Copies the debug and release AAR files into the Godot addon.
5. Downloads Godot and matching Android export templates.
6. Imports the Godot project.
7. Installs the Gradle Android build template.
8. Exports a debug APK.
9. Uploads the AARs and APK as a workflow artifact.

## Opening in Godot Android editor

The repository branch contains the full Godot project in `app/`. The compiled AAR is generated by CI and is intentionally not committed.

For direct phone-side APK export:

1. Download the latest workflow artifact.
2. Put the AARs into:
   - `app/addons/LocalAI/bin/debug/LocalAI-debug.aar`
   - `app/addons/LocalAI/bin/release/LocalAI-release.aar`
3. Open `app/project.godot`.
4. Enable the `LocalAI` editor plugin.
5. Install the Android build template.
6. Export using the `Android` preset with Gradle build enabled.

The GitHub-generated APK requires none of those manual steps.
